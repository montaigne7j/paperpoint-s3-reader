# V1.8.0 TTF memory and Paper S3 page-turn waveform notes

V1.8.0 combines the r13a/r14a/r15/r16 stabilization work into a GitHub release.

## FreeType / OpenFontRender allocator

OpenFontRender v1.2 normally initializes FreeType through `FT_Init_FreeType()`.  The release build uses `scripts/patch_openfontrender_ft_allocator.py` to replace that call with `CrossPointFtPsramAllocator::newLibrary()`, which creates the library with `FT_New_Library()` and a custom `FT_MemoryRec_` allocator.

Allocation policy:

- `size >= 512`: prefer `MALLOC_CAP_SPIRAM | MALLOC_CAP_8BIT`.
- `size < 512`: prefer `MALLOC_CAP_INTERNAL | MALLOC_CAP_8BIT`, then fallback to generic 8-bit heap.

Expected diagnostics:

```text
FT allocator summary: context=after-new-library ... psramAllocs=... internalAllocs=...
FT alloc: oldSize=0 newSize=... newLoc=PSRAM
```

## Background frame cache safety

Background frame-cache rendering is intentionally not allowed to rasterize new TTF glyphs.  It can draw RAM or persistent SD glyph-cache hits.  If drawing would require a new TTF rasterization, the frame-cache job is aborted and the framebuffer is not stored.

This prevents fallback/system glyphs from being persisted as a page cache and avoids background FreeType work fragmenting internal heap.

## Idle glyph prewarm

Reader idle time may prewarm a small number of missing glyphs for adjacent pages.  V1.8.0 keeps conservative defaults until the allocator behavior is verified:

- `internalFree >= 100000`
- `internalMaxAlloc >= 50000`
- `PSRAM free >= 3 MB`
- at most one glyph per pass

## Boot page-turn waveform settling

Paper S3 page-turn band-scan has separate darker-pass counts for boot-settle and stable phases:

- first 10 reader page turns: 4 target-black darker passes
- later page turns: 5 target-black darker passes

Expected diagnostic:

```text
Page-turn waveform profile: turn=1 settleTurns=10 blackDarkerPasses=4 profile=boot
Page-turn waveform profile: turn=11 settleTurns=10 blackDarkerPasses=5 profile=stable
```
